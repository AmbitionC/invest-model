### 1. 推送功能 ⏳ 缺失
**问题**: 有信号了怎么通知用户？
**需求**:
- [ ] 企业微信推送（免费、稳定）
- [ ] 短信推送（重要信号）
- [ ] 邮件推送（日报/周报）
- [ ] 推送条件配置（只推 strong_buy？还是所有信号变化都推？）

**实现方案**:
```python
# scripts/send_notification.py
- 检测信号变化（相比昨日）
- 生成推送内容（股票 + 信号 + 关键因子）
- 调用企业微信/短信 API 发送
```

**预计工作量**: 2-3 小时

### 2. 回测系统 ❌ 缺失
**问题**: 策略历史表现如何？能不能实盘？
**需求**:
- [ ] 历史信号回测（过去 5 年）
- [ ] 绩效统计（胜率、盈亏比、最大回撤）
- [ ] 基准对比（跑赢沪深 300 吗？）
- [ ] 参数优化（什么阈值最有效？）

**实现方案**:
```python

- 读取历史信号
- 模拟交易（买入→持有→卖出）

```

**预计工作量**: 2-3 天

### 3. 持仓管理 ❌ 缺失
**问题**: 买了多少、成本多少、盈亏多少？系统不知道
**需求**:
- [ ] 持仓记录（股票、数量、成本价）
- [ ] 盈亏计算（浮动盈亏、 realized PnL）
- [ ] 仓位控制（总仓位、单只上限）
- [ ] 操作记录（买入/卖出时间、价格）

**实现方案**:
```sql
-- 数据库表
CREATE TABLE positions (
    ts_code TEXT,
    quantity INTEGER,
    cost_price REAL,
    current_price REAL,
    pnl REAL,
    pnl_pct REAL
);

CREATE TABLE trades (
    trade_time TEXT,
    ts_code TEXT,
    operation TEXT,  -- buy/sell
    price REAL,
    quantity INTEGER,
    signal_id TEXT   -- 关联的信号
);
```

**预计工作量**: 1 天
