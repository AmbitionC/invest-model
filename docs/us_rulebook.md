# 美股模块规则手册（US Rulebook）

> 每条规则给出：参数、代码位置、方法论出处（life-teachers）。方法论迁移总纲：
> life-teachers `insights/us-stock-investing-methodology.md`（内含诚实声明：库内无美股专文，
> 期权造血为分析性迁移）。A 股规则见 `docs/rulebook.md`——两册独立，链路零交集。

## 架构隔离（不变式）

- 代码：`invest_model/us/` 独立包，不 import A 股因子/模型/组合/编排模块；只共用 `data`（schema/engine）与 `repositories.base`。
- 数据：`us_*` 前缀表，additive 建表，不触碰任何 A 股表。
- 运行：GitHub Actions `us-update.yml`（美东收盘后 22:10 UTC），**不进阿里云 FC**——A 股定时链路（live-watch/scheduler）零改动。
- 服务/前端：`/invest/us/*` 独立 API 前缀、独立页面。
- 输出性质：**纯建议（人工执行）**，尤其期权——V1 不做自动下单。

## 收益三来源标签（US 版）

每条建议标注赚哪种钱：**成长**（卫星/趋势内核心）、**修复**（恐慌窗买入）、**造血**（权利金现金流）。〔分析〕美股语境下第三来源从"分红"扩为"造血现金流"——中国税务居民股息预扣税使红利策略性价比低于 A 股。

## 明确不做（V1 负面清单）

- 自动下单/券商对接；裸卖期权；价差组合（spread）；日内交易；杠杆 ETF 持仓；
- Form 4 内幕跟庄、期权卖方历史回测——列为候选提案，数据成本评估后再议。
