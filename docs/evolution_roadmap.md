# Invest Journey · 自动化 / 可观测 / 可自我演进 —— 系统设计与路线

目标：一个「信号→计划→执行→复盘→规则迭代」全链路闭环的个人投资系统。
三大支柱与现状、下一步：

## 1. 自动化（Automation）

已就绪：
- 数据/计划/复盘/恐慌指数：阿里云 FC 定时任务（invest-scheduler），交易日自动跑批
- 信号入库：任何 `config/advisor_*.csv` / `holding_snapshot_*.csv` push 即幂等重灌
- 部署：push master 自动发布（fe-journey-faas 双 job：API + invest 定时函数）
- 风控执行建议：硬止损 / 破MA20 / 盈利保护(15/8/12) / 盈利后MA5/10梯子 / 再入场，
  全部由计划与盯盘自动产出带股数与挂单价的指令

下一步（按价值排序）：
- [ ] 持仓快照自动化：券商截图 → OCR → CSV 的半自动导入（或对接券商导出）
- [ ] 计划执行确认闭环：页面上对每条清仓/买入指令打勾回报，未执行项自动进入次日 risk_hints（后端字段已支持）
- [ ] 推送渠道升级：GitHub Issue 邮件 → 企业微信/飞书 webhook（盘中 crit 级即时达）

## 2. 可观测（Observability）

已就绪：
- `/invest/health` 系统脉搏：七条数据管道最新水位，落后行情日即红标（总览页）
- `/invest/shadow` 影子验证面板：研报速通 vs 旧闸门双轨虚拟净值（模型页）
- risk_hints 账户级警示：执行对账 / 行业>35%/单票>20% 集中度 / 仓位偏离 /
  排雷红旗（≥2 旗提示深挖）/ 戴维斯双杀预警（快报增速失速）
- 决策三层可解释（20260706）：因子归因（top_factors）→ 规则理由 → `docs/rulebook.md`
  出处（每条规则映射到 life-teachers 文章与验证报告）
- FaaS 告警 Issue（#11）：任务失败自动开票

下一步：
- [ ] 规则触发统计：各风控规则（硬止损/梯子/盈利保护/再入场）的月度触发次数与盈亏归因
- [ ] 数据水位跌破阈值时主动推送（而非等用户打开页面）

## 3. 可自我演进（Self-evolution）

已就绪：
- policy_shadow 双轨记录（每交易日自动更新），4~6 周后数据裁决「研报速通」保留或回退
  （RESEARCH_FAST_ENTRY=0 一键回退，无需改代码）
- 全部风控参数集中在 RiskConfig，回测口径与实盘口径同源（risk.py 纯函数）
- 验证方法论沉淀：入场规则漏斗 / 止盈参数网格 / 分年度稳健性 / 再入场 的研究脚本与结论
  （20260703 数据验证：严格闸门 2% 放行率、研报信号立即买 +12%、盈利后梯子回吐 19.8%→13.3%）

下一步：
- [ ] 投顾信号质量记分卡：按 source_type/时段/行业 统计信号 α 衰减曲线（需≥3个月信号积累）
- [ ] 参数月度再校准任务：滚动两年窗口重跑止盈网格，漂移超阈值时开 Issue 提议调参（人审后生效）
- [ ] 复盘报告自动引用影子验证与规则触发统计，形成「每周策略体检」

原则：任何规则变更必须过三关——历史回测、影子并行、一键回退。
