## 一、模块定位

`invest-model` 是 invest-journey 交易系统的**核心引擎**，负责：
- **数据层**：采集、存储、管理 A 股/ETF 多源异构数据
- **信号层**：基于多因子模型生成交易信号和置信度

## 四、数据流

```
1. 数据采集流程
   Tushare API → Collector → Validator → Repository → SQLite

2. 信号生成流程
   SQLite → Repository → Factor Calculator → Normalizer → Confidence → Signal

3. 信号输出
   Signal → Generator → (未来：推送层/风控层)
```

## 五、核心设计原则

### 5.1 数据层原则
- **数据源隔离**：Tushare/AkShare 通过统一接口访问，便于切换
- **模型驱动**：使用 dataclass 定义数据结构，类型安全
- **Repository 模式**：数据访问与业务逻辑分离
- **缓存优先**：减少 API 调用，提高效率

### 5.2 信号层原则
- **相对化**：所有指标必须相对化（分位数），不用绝对值
- **标准化**：统一输出 0-1 范围，便于合成
- **可追溯**：信号生成时保存因子快照，便于复盘
- **可配置**：因子权重、信号阈值通过配置文件调整

## 六、关键配置

### 6.1 因子权重

```yaml
factors:
  technical:
    weight: 0.30    # 技术面 30%
  fundamental:
    weight: 0.40    # 基本面 40%
  event:
    weight: 0.30    # 事件面 30%
```

### 6.2 信号阈值

| 置信度 | 信号类型 | 操作 |
|--------|---------|------|
| > 0.75 | 强买入 | 仓位 15-20% |
| 0.65-0.75 | 买入 | 仓位 10-15% |
| 0.55-0.65 | 观望 | 不操作 |
| 0.35-0.45 | 卖出 | 减仓 50% |
| < 0.35 | 强卖出 | 清仓 |

### 6.3 回溯期

- **技术因子**：250 天（1 年）
- **估值因子**：5 年（1250 天）
- **事件因子**：90 天（季度）
