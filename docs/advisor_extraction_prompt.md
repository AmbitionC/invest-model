# 投顾材料提炼规范（研报 / 早午盘 → 结构化信号）

本规范用于把投顾团队的**研报、每日早午盘点评**提炼成结构化信号，写入 `advisor_reco`
（个股）与 `advisor_theme`（行业/主题）两张表。当前采用**交互式提炼**（由 Claude 在会话中
读材料 → 按本规范产出 CSV → 人工确认 → `scripts/ingest_advisor.py` 入库）。本文同时是
将来切换「API 自动提炼」时的现成 prompt。

## 抽取四类信号

1. **个股 + 分级 + 催化剂**（→ `advisor_reco`）
2. **方向**：`long` 看多 / `reduce` 减持 / `avoid` 规避 / `exit` 逻辑证伪（触发逻辑止损清仓）
3. **行业/主题趋势**（→ `advisor_theme`）
4. **逻辑证伪/风险信号**：材料中明确「逻辑被破坏 / 政策叫停 / 证伪」→ `direction=exit`

## `advisor_reco` 字段填写

| 字段 | 规则 |
|---|---|
| `rec_date` | 材料日期 YYYYMMDD（生效起始日） |
| `code` | Tushare 代码，如 `300750.SZ` / `600519.SH`。**必须能在 `stock_info` 命中**，否则录入报错 |
| `source_type` | `research`（研报，中期有效）/ `intraday`（早午盘，短期，默认 3 个交易日后失效） |
| `grade` | `A` 产业拐点/重磅催化、强逻辑；`B` 业绩超预期/逻辑较强；`C` 题材/弱逻辑（默认不入仓） |
| `direction` | `long` / `reduce` / `avoid` / `exit` |
| `catalyst` | 一句话催化剂/推荐逻辑（用于「逻辑止损」对照） |
| `valid_until` | 失效日；留空：research=长期有效（持续到止损/换出），intraday=自动补 rec_date+N 交易日 |
| `source` | 团队/分析师标识 |
| `raw_excerpt` | 原文关键句留痕（审计用，可选） |

## 分级判据（A/B/C）

- **A 级**：产业趋势拐点、重磅政策/订单催化、龙头地位明确，逻辑链完整。→ 主仓重配。
- **B 级**：业绩超预期、景气改善、估值修复，逻辑较强但弹性/确定性次于 A。→ 中等仓位。
- **C 级**：纯题材、消息驱动、逻辑偏弱。→ 默认不纳入组合（`include_grade_c=False`）。

## 代码消歧规则

- 只给公司名时，按 `stock_info.name` 唯一匹配；同名/简称歧义时**列出候选请人工确认**，不臆造。
- 港股/美股/未在库标的：跳过并在确认环节说明（系统只交易 A 股 universe）。

## 提炼流程（交互式）

1. 用户发来研报 PDF / 早午盘文本。
2. Claude 按上表产出 CSV（reco 和/或 theme），**展示给用户确认**——A 级、看空/规避、重仓项必须确认。
3. 确认后运行 `scripts/ingest_advisor.py --kind reco --csv <file>` 入库。
4. 之后回测/操作计划自动读取当期有效信号。

## 风险与边界

- 从自然语言提炼交易信号有误读风险：**A 级 / 规避 / 重仓必须人工复核**。
- 投顾推荐的实际选股效果**无法历史回测**，只能上线后跟踪；本期回测只验证风控规则。
