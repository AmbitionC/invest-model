## 三、核心设计理念

### 3.1 相对化原则

**所有指标必须相对化，不能直接用绝对值！**

| 错误做法（❌） | 正确做法（✅） |
|--------------|--------------|
| `MACD > 0` 买入 | `MACD 分位数 > 0.8` 买入 |
| `RSI > 70` 卖出 | `RSI 分位数 > 0.9` 卖出 |
| `成交量 > 1 亿` 关注 | `量比 > 2` 关注 |
| `股价 > MA20` 买入 | `偏离度分位数 < 0.1` 买入 |

### 3.2 标准化方法

| 标准化方法 | 公式 | 适用场景 | 输出范围 |
|-----------|------|---------|---------|
| **分位数** | `percentileofscore(values, current) / 100` | 大多数指标 | 0-1 |
| **Z-Score** | `(x - mean) / std` | 正态分布数据 | -∞~+∞ |
| **Rank** | `rank / count` | 排名类数据 | 0-1 |

**默认使用分位数标准化**（对异常值不敏感）

### 4.2 布林带因子（Bollinger Bands）

**文件**: `signals/technical/bollinger.py`

| 指标名 | 计算公式 | 含义 | 信号阈值 |
|--------|----------|------|----------|
| position | (股价 - 下轨) / (上轨 - 下轨) | 股价在布林带中的位置 | >0.9 超买，<0.1 超卖 |
| bandwidth_percentile | 带宽在过去 250 天的分位数 | 波动率相对水平 | <0.1 变盘前兆 |

**信号逻辑**:
- 带宽收缩到历史 10% 分位 + 股价向上突破 → 买入
- 带宽扩张到历史 90% 分位 + 股价 > 上轨 → 卖出

**权重**: 5%

### 4.3 MACD 因子

**文件**: `signals/technical/macd.py`

| 指标名 | 计算公式 | 含义 | 信号阈值 |
|--------|----------|------|----------|
| DIF_percentile | DIF 在过去 250 天的分位数 | DIF 相对位置 | <0.2 超卖，>0.8 超买 |
| histogram_strength | \|MACD 柱\| / 过去 20 天最大\|MACD 柱\| | 动量强度 | >0.8 强动量 |
| divergence | 价格新高 but DIF 未新高 | 背离信号 | 1:顶背离，-1:底背离 |

**信号逻辑**:
- DIF 分位数 < 0.2 + 底背离 → 强买入

## 六、事件因子设计

### 6.1 股东减持因子

**文件**: `signals/event/holder_reduce.py`

| 减持类型 | 扣分 | 说明 |
|---------|------|------|
| 控股股东减持 | -15 分 | 强利空 |
| 高管集体减持（>3 人） | -10 分 | 中强利空 |
| 高管减持（1-2 人） | -5 分 | 中等利空 |
| 财务投资人减持 | -3 分 | 轻微利空 |

**信号逻辑**:
- 事件面基础分 0 分，根据减持类型扣分
- 扣分后事件面分 = max(-30, 基础分 - 扣分)

**权重**: 15%

- 事件面基础分 0 分，根据解禁情况扣分
- 扣分后事件面分 = max(-30, 基础分 - 扣分)

### 7.2 置信度公式

```python
# signals/composite/confidence.py
def 计算综合置信度 (股票数据：dict) -> float:
    """
    计算综合置信度（0-1 之间）
    
    置信度 = 0.5 + 基本面分 × 0.4 + 技术面分 × 0.3 + 事件面分 × 0.3
    
    基础分 0.5 表示中性起点
    """
    基础分 = 0.5
    
    # 基本面评分（-0.5 到 0.5）
    基本面分 = (
        估值评分 * 0.15 +
        成长评分 * 0.15 +
        盈利评分 * 0.10
    ) * 0.4
    
    # 技术面评分（-0.5 到 0.5）
    技术面分 = (
        布林带信号 * 0.05 +
        MACD 信号 * 0.05 +
        RSI 信号 * 0.05 +
        动量信号 * 0.05 +
        成交量信号 * 0.05 +
        波动率信号 * 0.05
    ) * 0.3
    
    # 事件面评分（-0.3 到 0）
    事件面分 = 0.0
    if 控股股东减持:
        事件面分 -= 0.15
    elif 高管减持:
        事件面分 -= 0.08
    elif 限售股解禁 (60 天内) and 解禁比例 > 0.1:
        事件面分 -= 0.10
    
    综合置信度 = 基础分 + 基本面分 + 技术面分 + 事件面分
    
    return max(0.0, min(1.0, 综合置信度))
```

### 7.3 信号阈值

| 置信度范围 | 信号类型 | 操作建议 |
|-----------|---------|---------|
| > 0.75 | 强买入 | 建议仓位 15-20% |
| 0.65-0.75 | 买入 | 建议仓位 10-15% |
| 0.55-0.65 | 观望 | 不操作 |
| 0.45-0.55 | 观望 | 不操作 |
| 0.35-0.45 | 卖出 | 减仓 50% |
| < 0.35 | 强卖出 | 清仓 |

## 九、信号生成器

### 9.1 信号数据结构

```python
# signals/generator.py
@dataclass
class 交易信号:
    """交易信号数据模型"""
    signal_id: str              # 信号 ID
    signal_time: datetime       # 信号产生时间
    code: str                   # 股票代码
    name: str                   # 股票名称
    signal_type: str            # buy/sell/watch
    confidence: float           # 置信度 0-1
    suggested_position: float   # 建议仓位%
    stop_loss: float            # 止损位
    target_price: float         # 目标位
    factors_snapshot: dict      # 因子快照（用于追溯）
    is_pushed: bool = False     # 是否已推送
    is_executed: bool = False   # 是否执行
```

### 9.2 信号生成流程

```python
def 生成信号 (股票代码：str) -> Optional[交易信号]:
    # 1. 获取数据
    日线数据 = 获取日线 (股票代码)
    财务数据 = 获取财报 (股票代码)
    事件数据 = 获取事件 (股票代码)
    
    # 2. 计算技术因子
    技术面分 = 计算技术面综合分 (日线数据)
    
    # 3. 计算基本面因子
    基本面分 = 计算基本面综合分 (财务数据)
    
    # 4. 计算事件因子
    事件面分 = 计算事件面综合分 (事件数据)
    
    # 5. 计算综合置信度
    置信度 = 0.5 + 基本面分 * 0.4 + 技术面分 * 0.3 + 事件面分 * 0.3
    
    # 6. 信号过滤
    if 置信度 < 0.6:
        return None
    
    # 7. 生成信号
    信号 = 交易信号 (
        signal_id=生成 ID(),
        signal_time=datetime.now(),
        code=股票代码，
        signal_type="buy" if 置信度 > 0.65 else "sell",
        confidence=置信度，
        suggested_position=计算建议仓位 (置信度),
        stop_loss=计算止损位 (日线数据),
        target_price=计算目标位 (日线数据),
        factors_snapshot={...}
    )
    
    return 信号
```
